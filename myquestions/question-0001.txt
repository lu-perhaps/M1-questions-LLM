El Problema:

En muchos datasets, varias filas pertenecen al mismo cliente, paciente o dispositivo. Si esas filas se reparten entre entrenamiento y validación, el modelo puede parecer mejor de lo que realmente es. 
Para evitar eso, necesitas evaluar un clasificador separando los datos por grupos.

Tu Misión:

Escribe una función llamada evaluar_modelo_por_grupos(df, group_col, target_col, n_splits=4) que:

1. Reciba un DataFrame de pandas con:
   - una columna de grupos definida por group_col
   - una columna objetivo binaria definida por target_col
   - una o más columnas numéricas predictoras

2. Separe las características X y la variable objetivo y, excluyendo de X las columnas group_col y target_col.

3. Use GroupKFold de sklearn con n_splits=n_splits para generar los folds, asegurando que las filas del mismo grupo nunca queden repartidas entre train y validación.

4. En cada fold:
   - entrene un modelo LogisticRegression(max_iter=1000)
   - realice predicciones sobre el fold de validación
   - calcule el accuracy del fold

5. Devuelva un diccionario con:
   - "accuracy_promedio": float con el promedio de accuracy de todos los folds
   - "accuracy_por_fold": np.ndarray con los accuracy de cada fold
   - "n_grupos": int con el número de grupos únicos en group_col

Argumentos:
- df (pd.DataFrame)
- group_col (str)
- target_col (str)
- n_splits (int, por defecto 4)

Retorna:
- dict

Restricciones:
- Debes usar pandas, numpy y sklearn
- Debes usar GroupKFold
- Solo debes usar como features las columnas numéricas distintas de group_col y target_col
